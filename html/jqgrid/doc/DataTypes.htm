When the first release of jqGrid was available the only possible way to obtain data was a xml way described in the example in Chapter 3.  Later many people request a JSON way, then Array data  and at end data with 'real' names. After lot of work and help of the community we have a wide range of obtaining data.
The related options (in options array) for manipulating different types of data are:

datatype: the possible options are - 'xml', 'json','clientSide' or 'local', 'xmlstring', 'jsonstring'

xmlReader : {
            root: "rows",
            row: "row",
            page: "rows>page",
            total: "rows>total",
            records : "rows>records",
            repeatitems: true,
            cell: "cell",
            id: "[id]",
            subgrid: {root:"rows", row: "row", repeatitems: true, cell:"cell"}
};
for xml data mapping

jsonReader : {
            root: "rows",
            page: "page",
            total: "total",
            records: "records",
            repeatitems: true,
            cell: "cell",
            id: "id",
            subgrid: {root:"rows", repeatitems: true, cell:"cell"}
}

for json mapping
In colModel the related options are xmlmap for description of xml field, and jsonmap for description of json field.  Example:
colModel : [ {name:'amaunt',..., xmlmap:'amt'...}...] will search in xml for  'amt' tag in the resulting xml when the repeatitems option is set to false.
