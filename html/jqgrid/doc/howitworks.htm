Understanding this will help you to work better with jqGrid and use the full capabilities of the plugin.

In all cases we should understand that we have two major divisions:

-         Server-side manipulation – I call it server layer and

-         Client side representation - I call it representation layer.

With other words jqGrid is component that helps you in easy way to represent database information at client side using a server-side technology. Moreover it helps you to manipulate that data back to the database.

What is server-side manipulation (SSM)? There are many definitions on that, but I try to explain it in terms of jqGrid.

Basically SSM means the server handles the editing and not the users’ browser. SSM isn’t something that is visible within a web page. Everything is done server side as with most programming languages. Basically it’s a server side command that tells the server to place a file or text within the page once it is called from a user.

In terms of jqGrid this means that you should care about this – you must have a piece of code that deals with information stored into the database using some scripting language and web server. Using this code you should return requested information back to the client (web browser). This is done with Ajax at client side. jqGrid uses Ajax calls to retrieve the requested information and represent it to client using Java Script language.

Having the needed (requested) information jqGrid construct the representation (tabular data) described from you in the so called column Model. To achieve a better representation jqGrid uses CCS (Cascading Style Sheet).

The constructed tabular data at client side has:

-         Caption layer
-         Header layer
-         Body layer
-         Navigation layer


Caption layer contain common information for the represented data.

Header layer contain information about the columns – labels, width and etc. Later we can see what we can do more with this.

Body layer is actually the requested data from the server.

Navigation layer contain additional information from the requested data and actions for requesting little pieces of information – in the literature called paging. Note that the navigation layer can be placed not only at bottom of the grid. Additionally you can place actions for editing, deleting, adding and searching data.

The minimum for the representing the data are Header layer and Body layer. Latter we will see haw we can achieve this.
