By default jqGrid expect 'xml' data to construct the grid. The default structure of this data is:

<?xml version ="1.0" encoding="utf-8"?>
<rows>
              <page> </page>
              <total> </total>
              <records> </records>
              <row id = “unique_rowid”>
                            <cell> cellcontent </cell>
                            <cell> <![CDATA[<font color=”red”>cell</font> content]]> </cell>
                            …
              </row>
              <row id = “unique_rowid”>
                            <cell> cellcontent </cell>
                            <cell> <![CDATA[<font color=”red”>cell</font> content]]> </cell>
                            …
              </row>
              …
</rows>

The encoding attribute in xml can be changed according to the encoded information.
<rows> tag  is the root tag for the grid.
<page> in this tag we return the number of the requested page.
<total> in this tag we return the total pages of the query.
<records> in this tag we return the total records from the query.
<row> in this tag we define a particular row in the grid.
<cell> in this tag is the actual data. Note that CDATA can be used. This way we can add images, links and check boxes. It is important to note that in this case the number of cell tags in the row tag should be equal to the number of cells in colModel. In our example we define six columns - so the number of cell tags in row tag should be equal to six.
It is important to note the id  attribute in <row> tag. In Principe this attribute can be omitted, but it is a good practice to have unique id of every row. If this attribute is omitted the jqGrid set his own id based on the order of the row. Some people asked me - haw to set the id without to view this id in the grid? The answer is simple. Do not include the unique id in colModel, but when we construct the query for the data we should include the unique id in the query and set them in the id attribute and not in the cell tag.
