Options
 Previous Top Next
  
Option Name
Description
url
Type : string
The url of the  file that holds the request . Default value: null
datatype
Type: string
Defines what type of information to expect to represent data in the grid. Valid options are
xml - we expect xml data
xmlstring - we expect xml data as string
json - we expect JSON data
jsonstring - we expect JSON data as string
clientSide - we expect data defined at client side  (array data)
Default value : xml
datastr
Type: string
The string of data when datatype parameter is set to xmlstring or jsonstring
Default value: null
mtype
Type: string
Defines the type of request to make ("POST" or "GET")
Default value: GET
colNames
Type: array
Array which describes the column labels in the grid
Default value: [] -empty array
colModel
Type: array
Array which describes the parameters of the columns. For a full description of all valid values see colModel API.
Default value: [] - empty array
xmlReader
Type: array
Array which describes the structure of the expected xml data. For a full description refer to Chapter 4 Data Types.
Default value:
{
            root: "rows",
            row: "row",
            page: "rows>page",
            total: "rows>total",
            records : "rows>records",
            repeatitems: true,
            cell: "cell",
            id: "[id]",
            subgrid: {root:"rows", row: "row", repeatitems: true, cell:"cell"}
}
jsonReader
Type: array
Array which describes the structure of the expected json data. For a full description refer to Chapter 4 Data Types.
Default value:
{
            root: "rows",
            page: "page",
            total: "total",
            records: "records",
            repeatitems: true,
            cell: "cell",
            id: "id",
            subgrid: {root:"rows", repeatitems: true, cell:"cell"}
}
height
Type: string
The height of the grid. Can be set as percentage or any valid measured value. 
Default value: 150px.
rowheight
Type: integer
This option defines the height of the single row . If is set the height of the grid is changed according to the number of the returned rows and the scroll bar does not appear
Default value: null
width
Type: number
If this option is not set, the width of the grid is a sum of the width’s of the columns defined in the colModel (in pixels). If this option is set the initial width of each column is set according to the value of shrinkToFit option. See below.
Default value: none
shrinkToFit
Type: boolean
This option describes the type of calculation of the initial width of each column against with the width of the grid. If the value is true and the value in width option is set then:
Every column width is scaled according to the defined option width. Example: if we define two columns with a width of 80 and 120 pixels, but want the grid to have a 300 pixels - then the columns are recalculated as follow: 1- column = 300(new width)/200(sum of all width)*80(column width) = 120 and 2 column = 300/200*120 = 180.  The grid width is 300px.
If the value is false and the value in width option is set then:
The width of the grid is the width set in option. The column width are not recalculated and have the values defined in colModel.
Default value: true
page
Type: integer
The requested initial page number when we use datatypes xml or json (data returned from server)
Default value: 1
sortorder
Type: string
The initial  sorting order when we use datatypes xml or json (data returned from server)
Default value: asc
sortname
Type: string
The initial  sorting name when we use datatypes xml or json (data returned from server)
Default value: empty string
rowNum
Type: integer
The initial number of rows that must be returned from the server
Default value: 20
pager
Type: DOM element
Sets the pager bar for the grid. Must be a valid html element. If the element has class “scroll”, then the width is equal to the grid.
rowList
Type: array
This parameter construct a select box element in the pager in which the user can change the number of the visible rows. Default value: [] - empty array.
viewrecords
Type: boolean
Display the total records from the query in the pager bar.
Default value: false
recordtext
Type: string
Displays a text before the viewed total records.
Default value: Rows
firstimg , previmg , nextimg, lastimg
Type: string
Liks to image url which are used in the pager bar for the first, previous, next and last buttons.
Default value: first.gif, prev.gif, next.gif, last.gif
sortascimg, sortdescimg
Type: string
Liks to image url which are used when the user sort a column.
Default value: sort_asc.gif, sort_desc.gif
loadtext
Type: string
The text which appear when requesting and sorting data.
Default value: Loading...
altRows
Type: boolean
Set a zebra like grid.
Default value: true
imgpath
Type: string
Defines the path to the images that are used in the grid. Set this option whit out / at end.
Default value: '' - empty string
caption
Type: string
Defines the Caption layer for the grid. This caption appear above the Header layer. If the string is empty the caption does not appear.
Default value: '' - empty string
hidegrid
Type: boolean
Enables or disables the hide grid button, which appear into the right side of the Caption layer. Have sense only if caption parameter is not empty string.
Default value: true
loadonce
Type: boolean
If this flag is set to true, the grid load only once the data from the server according to the parameter datatype. After the first request (xml or json) the datatype parameter is set automatically to clientSide. All other manipulations are done at client side. The manipulations at pager (if any) are disabled.
Default value: false
multiselect
Type: boolean
If this flag is set to true a multi selection of rows is enabled. A new column at left side is added. Can be used with any datatype option.
Default value: false
multikey
Type: string
This parameter have sense only multiselect option is set to true. Defines the key which will be pressed when we make multiselection. The possible values are:
shiftKey - the user should press Shift Key
altKey - the user should press Alt Key
ctrlKey - the user should press Ctrl Key
Default value: '' empty string.
subGrid
Type: boolean
Is set to true this enables using of sub grid. Add additional column at the left side of the grid. For more information see Setting subgrid
Default value: false
subGridUrl
Type: string
This option has effect only if subGrid option is set to true. This describes the url for the subgrid data. Additinally to this url is added parameter “id” which is the id of the row.
Default value: '' empty string
subGridModel
Type: array
This option has effect only if subGrid option is set to true. This option describes the model of the subgrid. For more details see Chapter XX
Default Value: [] - empty array
editurl
Type: string
Defines the url for inline and form editing.
Default value: null
onSelectRow
Type: function
Parameter: rowid - the id of the selected row
Raised immediately after row was clicked.
Calling convention:
onSelectRow: function( rowid) {
// here is the code
}
Default: null
ondblClickRow
Type: function
Parameter: rowid - the id of the selected row
Raised immediately after row was double clicked.
Calling convention:
ondblClickRow: function( rowid) {
// here is the code
}
Default: null
onRightClickRow
Type: function
Parameter: rowid - the id of the selected row
Raised immediately after row was right clicked.
Calling convention:
onRightClickRow: function( rowid) {
// here is the code
}
Default: null
onSortCol
Type: function
Parameters:
index - the index name from colModel
colindex - the index of column
Raised immediately after sortable column was clicked and before sorting the data
Calling convention:
onSorCol: function( index, colindex) {
// here is the code
}
Default: null
onPaging
Type: function
Parameters: none
This event fires after click on page button and before data population. Also works when the user enter new page number in the page input box.
loadComplete
Type: function
Parameters: none
This event is executed immediately after every server request
Default: null
subGridRowExpanded
Type: function
Parameters
pID - the unique id of the div element where we can put contents when subgrid is enabled
id - the id of the row
This even raises when subgrid is enabled and is executed when the user click on the plus icon of the grid. Can be used to put custom data in the subgrid.
Default: null
subGridRowColapsed
Type: function
Parameters
pID - the unique id of the div element where we can put contents when subgrid is enabled
id - the id of the row
This event raises when the user click on the minus icon.
Default: null
postData
Type: array
This array is passed directly to the url. This is associative array and can be used this way: {name1:value1...}. See API methods for manipulation.
Default: empty array
userData
Type: array
This array contain custom information from the request. Can be used at any time. See API methods for manipulation
Default: empty array.
toolbar
Type: array
This option defines the toolbar of the grid. This is array with two values in which the first value enables the toolbar and the second defines the position relative to body Layer. Possible values "top" or "bottom"
Default; [false,"top"]
$.jgrid.default
Type: array
This array is used to define a common grid option in the application. Typically this can be called once to set a common grid parameters. Note that all parameters can be changed. Typical implementation; $.expend($.jgrid.defaults,{rowNum:10})
