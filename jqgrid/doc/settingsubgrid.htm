Setting subgrid
 Previous Top Next
When setting the subgrid  a three option should be considered.

subGrid this boolean value enables or disables the subgrid. If the grid is enabled a additional column at left side is added to the grid. This column contain a 'plus' image which indicate that the user can click on it to expand the row. By default all rows are collapsed.

Default value for this option is false. To enable a subgrid set it to true.


subGridUrl - this option points to file that from which we should get the data for the subgrid. Additionally to this url a id is passed as parameter. The value is the id of the row. If there is a need to pass additional  parameters we can use a params option in subGridModel (see below)

subGridModel  is a array in which we describes a column model for the subgrid data. The syntax is

subGridModel : [
{ name : ['name_1','name_2',...,'name_n'],
width : [width_1,width_2,...,width_n] ,
params : [param_1,...,param_n]} 
]

where
name  is array that contain the labels of the columns of the subgrid.
width is array that contain the width of the column. This array should have the same length as in name array.
params is array in which we can add a name from colModel to pass as additional parameter to the subGridUrl.


Again whit these options we must pay attention to subgrid model.
The subgrid model is described in xmlReader and jsonReader.

xmlReader : {
root: "rows",
row: "row",
page: "rows>page",
total: "rows>total",
records : "rows>records",
repeatitems: true,
cell: "cell",
id: "[id]",
subgrid: {root:"rows", row: "row", repeatitems: true, cell:"cell"}
}

for xml mapping.

jsonReader : {
root: "rows",
page: "page",
total: "total",
records: "records",
repeatitems: true,
cell: "cell",
id: "id",
subgrid: {root:"rows", repeatitems: true, cell:"cell"}
}

The mapping rules are the same as these in the basic grid.
