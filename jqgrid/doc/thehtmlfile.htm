Below I will explain every line of code:

I suppose that the saved file is in the directory where you upload the jqGrid files.
First it is needed to include the needed files to construct the grid. This is done between the head tags in the html document.

<head>
<link rel="stylesheet" type="text/css" media="screen" href="themes/basic/grid.css" />
<script src="jquery.js" type="text/javascript"></script>
<script src="jquery.jqGrid.js" type="text/javascript"></script>
…
</head>
the <link…/>  tag load the style sheet for jqGrid.
The first <script ../> tag load the jquery library, the second loads the needed jqGrid plugins for use and in the third script tag we write the needed commands to construct the grid. See detailed description of this below.
Between body tags you should define where you want to place the grid.
<body>

The definition of the grid is done via the html tag <table>. To make our life easy it is good idea to give the table unique id in the html document. The second step is to assign a class “scroll” which means that we want to use the CSS provided with a grid. We want to use a paging mechanism too, so we should define the navigation layer. This can be done with a most used tag div. Giving the class “scroll” of the navigator tells that we want to use the provided CSS of the jqGrid. It is important to note that the navigation layer can be placed in arbitrary place of the html document. In this case it is under the grid.
As you know in most cases juery uses document.ready function to do the needed. For more information refer to jquery documentation. The syntax of constructing the grid is:

jQuery('#grid_selector').jqGrid( options )

where:
'#grid_selector' is the unique id of the grid table

jqGrid is the plugin and

options is a array of the needed information to construct the grid. Let’s begin with the options array which looks like this: (The order of option can be in arbitrary way)

{
url:'example.php',
datatype: 'xml',
mtype: 'GET',
colNames:['Inv No','Date', 'Amount','Tax','Total','Notes'],
colModel :[
{name:'invid',index:'invid', width:55},
{name:'invdate',index:'invdate', width:90},
{name:'amount',index:'amount', width:80, align:'right'},
{name:'tax',index:'tax', width:80, align:'right'},
{name:'total',index:'total', width:80,align:'right'},
{name:'note',index:'note', width:150, sortable:false} ],
pager: jQuery('#pager'),
rowNum:10,
rowList:[10,20,30],
sortname: 'id',
sortorder: 'desc',
viewrecords: true,
imgpath: 'themes/basic/images',
caption: 'My first grid'
}
url: option tell us from where to get the needed data. Typically this file have connection to database and return the appropriate information which will be filled into the Body layer in grid;
datatype:  option tells jqGrid what is the type of information to construct the grid. In this case we tell the grid that we expect xml data returned from the server. For a list of all available datatypes refer to full API docs. By default this option is set to 'xml';
mtype: option tells us haw to make the ajax call - with 'GET' or 'POST' method. In this case we will use GET method, which is default;
colNames: is  array in which we describe the names  in the columns. This is the text that apper in the head of the grid (Header layer). The names are separated with comma;
colModel: array that describes the model of the columns. This is the most important part of the grid. Here briefly I will explain the available options. For the complete list of options see API documentation:
              name: is the name of the column,  (It is not obligatory that this name is the name from database table, but later we will see haw we can use tis when we have different data formats)
              index: is the name passed to the server to sort data note that we can pass here numbers too.  Typically this is the name from database. Here we can put more than one name.
              width: is the width of the column 
              align: is the align of the column (default is left)
              sortable: defines if this column can be sorted (default true)  ;
rowNum: parameter describes how many records we want to view in the grid.  This parameter is passed to the url;
pager:  pager parameter define that we want to use a pager bar to navigate through the records . In this case this must be a valid html element. Note that the pager can have a position where you want. In this case we want that the pager should after the Body layer. When we give the pager the class 'scroll', then this element will use the predefined style from grid style sheet;
rowList:  array parameter construct a select box element in the pager in which we can change the number of the visible rows. When changed during the execution this parameter replaces the rowNum parameter that is passed to the url;
sortname: sets the initial sorting column. Can be a name or number. This parameter is added to the url;
sortorder: sets the sorting order. Default is asc. This parameter is added to the url;
viewrecords: Display the number of total records from the query in the pager bar. Default is false;
imgpath: path to image location needed for the grid. The path should not end with '/'. It is important to note, that this parameter should be set in appropriate way;
caption: sets the caption for the grid. If this parameter is not set the Caption layer will be not visible;

Having this we do the half of work. The next step is to construct the Server side manipulation which is done in the in file pointed in url parameter in grid
