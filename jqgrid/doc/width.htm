The width of the grid can be controlled different ways and is set only in pixels.
Related options in options array:
width
shrinkToFit
Related option in colModel - width

By default width parameter in options array is not set, which means that the width of the grid is calculated as sum of the width parameter in colModel. If the width parameter in colModel is not set the default width is 150.
Before some  time one guy asked me - why the width of the grid is not equal to the sum of the width in colModel? The sum is 500px, but I have 515px!

The answer for me was not easy :). After some investigations I discover that we should pay attention of the CSS and what is set for that elements in the CSS.

The true answer is - the width of grid is equal to the sum of the width in colModel +
padding and border settings for the cell in CSS.

That is: if we have 5 colums which sum is 500 and in the CSS we have the flowing
definition for the table data:

table.scroll tbody td  {
    padding: 2px;
    text-align: left;
    border-bottom: 1px solid #D4D0C8;
    border-left: 1px solid #D4D0C8;
    text-overflow: ellipsis;
    overflow: hidden;
    white-space: nowrap;
}

the grid width will be 500 + 5*(2 padding pixels + 1 border left pixel) = 515
In other words we should add extra 3 pixels for every column - 2 padding and 1 border-left.
In some cases it is needed to have fixed width independent of the width set in colModel.

Here we can play with the options in options array - width and shrinkToFit.
The default value of shrinkToFit is true which means that if we set the width parameter all of the initial width of columns are recalculated as:

width/swidth*colModel.width

where

width is width that we want to have
swidth is the sum of width set in colModel
colModel.width is the width for the column.

If the value of shrinkToFit is false then we do not make any recalculation of the initial column width and grid will have the  width set in options array.
